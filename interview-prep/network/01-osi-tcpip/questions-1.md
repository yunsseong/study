# OSI 7계층 / TCP/IP 면접 질문

> 질문을 보고 직접 답변해보세요. 답변은 01-osi-tcpip-answers.md에서 확인할 수 있습니다.

---

## 기본 개념

**Q1.** OSI 7계층이 무엇이고, 왜 계층을 나누나요?
네트워크를 7계층으로 나눈 표준 규약입니다.
7계층으로 나눈 이유는, 네트워크를 하나의 거대한 모델로 적용했을때, 문제가 발생하면 어느 계층에서 발생한지 파악이 어렵습니다. 또한 하나의 변경이 전체에 영향을 미치기 쉽습니다. 그래서 역할에 따라 7계층으로 나눴습니다.

**Q2.** OSI 7계층의 각 계층 이름과 역할을 설명해주세요.
1계층은 물리 계층이고 0과 1을 전달하는 역할을 합니다.
2계층은 데이터 링크 계층이고 같은 네트워크 내에서의 전송을 담당 합니다.
3계층은 네트워크 계층이고 다른 네트워크 간의 전송을 담당합니다.
4계층은 전송 계층이고 프로세스 간의 전송을 담당합니다.
5계층은 세션 계층이고 세션 유지 및 관리를 담당합니다.
6계층은 표현 계층이고 데이터 암호화를 담당합니다.
7계층은 응용 계층이고 사용자가 주로 사용하는 애플리케이션을 담당합니다.

**Q3.** TCP/IP 4계층을 설명하고, OSI 7계층과의 매핑 관계를 말해주세요.

TCP/IP는 OSI를 간략화하여 실제 실무에서 사용하는 모델입니다.
OSI 1,2계층이 TCP/IP 1계층인 네트워크 액세스 계층과 대응되고
2계층이 인터넷 계층과 대응되고
3계층이 전송 계층과 대응되고
4~7계층이 응용 계층과 대응됩니다.


**Q4.** 캡슐화(Encapsulation)와 역캡슐화(Decapsulation)를 설명해주세요.
계층을 내려갈때마다 헤더가 붙는데 이를 캡슐화라고 하며,
계층을 올라갈때마다 헤더가 탈락되는데 이를 역캡슐화라고 합니다.

**Q5.** PDU(Protocol Data Unit)가 무엇인가요? 각 계층별 PDU 이름을 말해주세요.
각 프로토콜에서 다루는 데이터의 단위이며,
2계층은 프레임
3계층은 패킷
4계층은 세그먼트
7계층은 데이터 입니다.

## 비교/구분

**Q6.** 2계층(데이터링크)과 3계층(네트워크)의 차이는 무엇인가요?
데이터링크 계층은 같은 네트워크 사이의 통신을 담당하고, 네트워크 계층은 다른 네트워크 사이의 통신을 담당합니다.

**Q7.** MAC 주소와 IP 주소의 차이를 설명해주세요.
MAC은 네트워크 카드에 부여된 불변의 주소라면 IP는 동적으로 바뀔 수 있는 주소입니다.

**Q8.** 3계층(네트워크)과 4계층(전송)의 차이는 무엇인가요? IP와 TCP의 역할 차이로 설명해주세요.
네트워크 계층은 다른 네트워크 간의 통신을 통해 도착지 컴퓨터까지 도착 하는 것을 역할로 합니다.
전송 계층은 컴퓨터 내부에서 프로세스까지 데이터를 전달하는 것이 역할입니다.

**Q9.** OSI 모델과 TCP/IP 모델은 왜 다른가요? 실제 인터넷은 어떤 모델을 따르나요?
OSI 모델을 그대로 사용하기에는 너무 세분화되어 있어서 이를 간략화한 TCP/IP 모델을 사용합니다. 실제 인터넷은 TCP/IP 모델을 사용합니다.

## 심화/실무

**Q10.** 브라우저에서 Spring Boot 서버로 요청이 갈 때, 각 계층에서 어떤 일이 일어나는지 설명해주세요.
7계층 브라우저에서 스프링 서버로 요청을 보내면
4계층에서 TCP 세그먼트로 변환이되고
3계층에서는 DNS를 통해 상대방의 IP를 알아내어 IP 패킷으로 변환되고
2계층에서는 프레임으로 변환되고
1계층에서 전기신호로 스프링 서버로 전송됩니다.

스프링 서버에서
1계층에서 신호를 받아
2계층에서 프레임을 열어 본인 MAC 주소인지 확인합니다.
3계층에서 패킷을 열어 본인 IP 주소와 일치하는지 확인합니다.
4계층에서 포트를 보고 해당 하는 프로세스로 전달합니다.
7계층에서 스프링 서버의 DispatcherServlet이 이를 받아 적절한 컨트롤러에게 넘겨줍니다.

**Q11.** 라우터와 스위치는 각각 몇 계층 장비인가요? 차이를 설명해주세요.
스위치는 2계층 장비이며, 라우터는 3계층 장비 입니다.
스위치는 MAC 주소를 기반으로 데이터를 전송하고, 라우터는 IP 기반으로 도착지까지 최적의 경로를 라우팅합니다.

**Q12.** HTTP는 몇 계층 프로토콜인가요? 그 아래 계층에서는 무엇이 동작하나요?
HTTP는 7계층 프로토콜 입니다. 그 아래 계층인 4계층에서는 TCP 혹은 UDP 프로토콜이 동작합니다.

## 꼬리질문

**Q13.** "계층을 나누면 유지보수가 쉽다"고 했는데, 구체적인 예시를 들어주세요.
SSL 버전을 새로 추가했을때, 계층이 하나라면 이에 영향 받는 부분을 모두 수정해야하지만, 계층을 나눠서 SSL을 담당하는 표현 계층만 수정하면 됩니다.

**Q14.** 5, 6계층(세션, 표현)은 실무에서 왜 잘 언급되지 않나요?
사용자가 직접 개입하는 것이 아닌 규격에 따라 자동으로 동작하는 부분이기 때문에 거의 실무에서 잘 언급되지 않습니다.

**Q15.** IP 주소만으로는 왜 통신이 불완전한가요? 포트 번호가 왜 필요한가요?
IP 주소는 컴퓨터까지는 접근이 가능하지만, 컴퓨터 내부에 동작하고 있는 어떤 프로세스에 전달해야할지에 대한 정보가 없어 불완전합니다.
