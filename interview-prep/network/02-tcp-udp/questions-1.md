# TCP vs UDP 면접 질문

> 질문을 보고 직접 답변해보세요. 답변은 answers.md에서 확인할 수 있습니다.

---

## 기본 개념

**Q1.** TCP와 UDP의 차이점을 설명해주세요.
TCP는 연결 지향 프로토콜로 3way hand shake를 통해 연결을 확보하고 각 패킷마다 ACK로 제대로 패킷이 도달했는지 확인합니다.
UDP의 경우 비연결 프로토콜로 이러한 과정 없이 바로 데이터를 전송하는 것에 차이가 있습니다. 이러한 점 덕분에 TCP보다 속도가 빠르다는 장점이 있지만 데이터가 유실될 가능성이 있습니다.

**Q2.** TCP의 신뢰성이란 구체적으로 무엇을 의미하나요?
데이터가 반드시 목적지까지 전송되는 것을 의미합니다. 데이터를 전송한 후 ACK 신호를 상대방으로부터 받아 데이터가 제대로 전송되었는지 확인하고,
ACK 응답이 시간 초과될 경우 다시 재전송하여 신뢰성을 확보합니다.

**Q3.** TCP의 흐름 제어(Flow Control)와 혼잡 제어(Congestion Control)의 차이를 설명해주세요.
흐름 제어는 상대방 컴퓨터가 처리할 수 있는 용량에 맞춰 데이터의 양을 조절하여 보내는 것이고,
혼잡 제어는 현재 네트워크에 많은 패킷이 돌아다녀 혼잡할때 전송하는 데이터 양을 낮춰서 보내는 것을 의미합니다.

**Q4.** UDP는 신뢰성을 보장하지 않는데 왜 사용하나요?
UDP는 TCP 보다 더 빠르게 데이터를 전송할 수 있습니다. 신뢰성이 매우 중요하진 않지만 실시간성이 필요한 서비스에 사용하는 것이 적합합니다.
예를 들면 음성 통화, 라이브 스트리밍 등이 있습니다.

**Q5.** TCP와 UDP의 헤더 크기 차이는 왜 생기나요?
TCP에는 신뢰성과 순서를 보장하기 위한 데이터가 헤더에 있습니다. 예를들면 시퀀스 넘버가 있습니다. 하지만 UDP는 신뢰성과 순서를 보장할 필요가 없기 때문에
헤더에 이런 정보들을 기입하지 않아도 되어 TCP보다 UDP 헤더의 크기가 더 작습니다.

## 비교/구분

**Q6.** TCP를 사용하는 프로토콜과 UDP를 사용하는 프로토콜을 각각 3개씩 말하고, 왜 그 프로토콜을 쓰는지 설명해주세요.
HTTP, SSH, FTP 등이 있습니다. 해당 프로토콜은 신뢰성이 중요하기 때문에 TCP 프로토콜을 사용하여 신뢰성을 보장합니다.

**Q7.** 온라인 게임은 왜 UDP를 쓰나요? 패킷이 유실되면 어떻게 하나요?
온라인 게임의 경우 현재 데이터가 과거 데이터보다 더 중요하고 실시간성이 중요하기 때문에 유실되어도 괜찮으며 속도가 빠른 UDP를 사용합니다.
패킷이 유실되더라도 다음 패킷이 사람이 인지하는 속도보다 더 빠르게 도착하기 때문에 문제가 되지 않습니다.

**Q8.** DNS는 왜 UDP를 쓰나요?
dns 데이터 자체가 작고, 또한 응답 속도가 중요하기 때문입니다.

**Q9.** 세그먼트와 데이터그램의 차이는 무엇인가요?
세그먼트는 TCP의 PDU이고 데이터 그램은 UDP의 PDU 입니다.

## 심화/실무

**Q10.** HTTP/3는 왜 TCP가 아닌 UDP 기반(QUIC)을 사용하나요?
UDP로 신뢰성을 확보할 수 있기 때문에 TCP가 아닌 UDP 기반 (QUIC)를 사용하여 구동합니다.

**Q11.** UDP인데 신뢰성이 필요하면 어떻게 해결하나요?
애플리케이션 레이어에서 신뢰성을 확보하는 기능을 구현하여 구동하면 됩니다. 예를들면 QUIC가 있습니다.

**Q12.** TCP는 왜 느린가요? 구체적으로 어떤 오버헤드가 있나요?
3way hand shake와 tcp 헤더, 혼잡 제어, 흐름 제어, ack 응답 받기, 재전송 등이 있습니다.

**Q13.** Spring Boot는 왜 TCP를 사용하나요? UDP를 쓰면 안 되나요?
신뢰성이 중요하기 때문에 TCP를 사용합니다. Json 데이터를 전송하는데, 일부라도 누락이되면 구조가 깨져 데이터를 사용하지 못하는 경우가 발생할 수 있기 때문입니다.

## 꼬리질문

**Q14.** TCP의 순서 보장은 어떤 메커니즘으로 동작하나요?
TCP 헤더 속 시퀀스 넘버를 통해서 순서를 보장합니다.

**Q15.** "연결 지향"이란 구체적으로 무엇을 의미하나요? 물리적 연결인가요?
연결 지향은 먼저 연결을 맺고 데이터를 보내는 것을 의미합니다.
