## 문제 1: 호이스팅 동작 이해

### 예측
```
a: undefined 로 나올 것이다. 
b: ReferenceError가 나올 것이다.
```

### 실제 결과
```
a: undefined
b: ReferenceError
```

### 설명
var는 호이스팅이 가능하나, 선언만 호이스팅되고 할당은 원래 자리에서 이뤄지기 때문에 undefined가 나온다.
let도 호이스팅이 가능하지만, Temporal Dead Zone에 의해서 오류가 발생한다.

## 문제 5: 블록 스코프 이해

### 예측
```
블록 안 x: 3
블록 안 y: 4
블록 밖 x: 1
블록 밖 y: 2
블록 밖 z: 5
```

### 실제 결과
```
블록 안 x: 3
블록 안 y: 4
블록 밖 x: 1
블록 밖 y: 2
블록 밖 z: 5
```

### 설명
[let/const의 블록 스코프와 var의 함수 스코프 차이를 설명해주세요]
let/const의 블록 스코프는 { } 단위로 같은 변수이냐 아니냐 나뉘게 되고, var의 함수 스코프는 함수 단위로 같은 변수냐 아니냐 나뉘게 됨
